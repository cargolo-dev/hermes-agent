# CARGOLO ASR Project Context

This repository contains a production-oriented MVP scaffold for a Hermes-based operations system for CARGOLO ASR (Air/Sea/Rail).

Core architectural rules:
- Hermes does not receive mail directly; n8n prefilters and normalizes email events, then sends them via webhook.
- The webhook ingress already exists and remains the event boundary.
- The local order folder is the operative case file, audit trail, and document store.
- Every new email must be compared against the existing case file and the latest TMS snapshot before any recommendation is produced.
- No automatic customer sending.
- No destructive actions.
- No uncontrolled TMS writes.
- All customer-visible responses are drafts only.
- All significant actions must be audit-logged.
- Idempotency is mandatory.
- Secrets only via environment variables or Hermes runtime config, never hardcoded in committed files.

Implementation layout:
- plugins/cargolo_ops/: deterministic domain logic (models, storage, adapters, processing, reporting)
- tools/cargolo_asr_tool.py: Hermes tool surface for processing events, pulling mail history, generating reports
- docs/cargolo-asr/: architecture and workflow artifacts
- tests/cargolo_ops/: regression coverage for schemas, processing, idempotency, reporting

Runtime storage default:
- ~/.hermes/cargolo_asr/
  - orders/AN-xxxxx/
  - review_queue/
  - mock_tms/

Expected env vars:
- HERMES_CARGOLO_ASR_MAIL_HISTORY_URL
- HERMES_CARGOLO_ASR_MAIL_HISTORY_TOKEN (optional)
- HERMES_CARGOLO_ASR_MAIL_HISTORY_TIMEOUT (optional)

Current webhook best practice:
- webhook route receives raw JSON body
- HMAC in X-Hub-Signature-256
- n8n sends top-level mail_context
- ASR ingest route uses a deterministic direct processor before the LLM summary step
- when an AN has no local case folder yet, the first processing pass should initialize the case by pulling mail history first and writing a TMS snapshot before recommendations are emitted
- Hermes/agent should prefer deterministic tool processing over nested prompt-field parsing where possible
